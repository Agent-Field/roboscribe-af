# CLAUDE.md — OXE-Narrator project guide

Per-project guidance for AI coding agents extending this example. Read this BEFORE editing — it captures the architectural invariants the skill (`agentfield-multi-reasoner-builder`) enforces.

## What this project is

A hierarchical language annotator for robotics episodes (PushT v1, LeRobot/OXE planned v1.5), built as a multi-reasoner system on AgentField with Deep Lake as the multimodal substrate. Produces structured `EpisodeAnnotation` objects: episode-grain goal + outcome, sub-task segments with verb-phrase labels, per-modality phase tags, cross-modal consistency signals.

## Non-negotiable invariants

These come straight from the AgentField skill. Violating them turns a 16-reasoner composite back into a chain framework with extra ceremony.

| Invariant | What it means here |
|---|---|
| Every reasoner has a one-sentence contract | Each `.ai()` reasoner answers ONE cognitive question. `object_detector` = "what's in this frame?", `boundary_judger` = "which candidates are real sub-task boundaries?". If you can't write it in one sentence, decompose further |
| Flat output schemas (2-4 attributes) | See `schemas.py`. `DetectedObjects` has 2 fields. `MotionPhase` has 3. `EpisodeAnnotation` is *assembled* from many flat schemas — never one giant schema generated by one call |
| `confident: bool` on every `.ai()` schema | Every model output declares its own confidence; quality_signals downstream aggregates these into `human_review_recommended` |
| `model: str \| None = None` propagated through every reasoner | Per-request model override is mandatory. `app.call(..., model=model)` everywhere |
| Async parallelism wherever independent | `asyncio.gather` is used at 4 layers: visual_thread || action_thread, scene_describer's per-keyframe object_detector, temporal_segmenter's per-segment narrators, cross_modal_verifier || episode_goal_synthesizer |
| Deterministic work in `@skill()` | `velocity_profile`, `visual_action_aligner`, `load_episode_keyframes` — no LLM. The bias is to compute what code can compute |
| Strings between LLMs, JSON only for code routing | When `action_thread` hands a velocity summary to `trajectory_phase_classifier`, it passes a prose string built by the skill — not a list of floats. Same on the temporal layer |
| Cross-boundary data does NOT auto-reconstitute | `app.call` returns `dict` regardless of receiver type hints. Reconstruct Pydantic models explicitly: `VisualThreadResult(**visual_dict)` |
| Depth ≥ 3 layers from entry | `annotate_episode → visual_thread → scene_describer → object_detector` = depth 4. Stay deep |
| At least one place where graph shape depends on intermediate state | `temporal_segmenter` decides at runtime how many `segment_narrator` instances to spawn, based on what `boundary_judger` returned. This is the meta-prompting pattern — don't lose it |

## What's a reasoner vs a skill

```
@router.reasoner()  → cognitive unit. Calls .ai() and/or .call(). MAY have judgement
@router.skill()     → deterministic. No LLM. Same inputs → same outputs. Use whenever a `for` loop could do it
```

Both register with the control plane, both are callable via `app.call`, both appear in the workflow DAG. The decorator declares intent.

## What's NOT in this project (by design)

- **HUNT → PROVE adversarial pattern.** Annotation is not a binary-verdict problem. The cross-modal verifier is structural verification (do two independent modality experts agree?), not adversarial. Don't add an adversary "for safety" — it would double cost without earning its keep
- **Large monolithic `analyze_everything()` reasoner.** If you find yourself writing one, you decomposed wrong
- **Direct HTTP between reasoners.** Always `app.call(f"{NODE_ID}.target")`. The control plane needs to see every edge for the workflow DAG
- **`app.harness()`.** Not needed — `app.ai(tools=[...])` covers stateful tool-using, and we don't need a coding-agent CLI in this pipeline. The Docker image stays slim

## Adding a new task adapter (e.g. Aloha)

```python
# tasks/aloha.py
from .base import EpisodeFrames, EpisodeActions, TaskAdapter
from .registry import register_adapter

ALOHA_DESCRIPTION = "Aloha is a bimanual..."

class AlohaTransferAdapter:
    name = "aloha_transfer"
    source_dataset_id = "lerobot/aloha_sim_transfer_cube_human"
    description = ALOHA_DESCRIPTION
    embodiment = "bimanual ViperX-300 14-DoF"
    relevant_scorers = ["smoothness", "visual_quality", "task_success", "novelty", "edge_case"]

    def load_keyframes(self, episode_id, n=4): ...
    def load_actions(self, episode_id): ...
    def n_episodes(self): ...

register_adapter("aloha_transfer", lambda: AlohaTransferAdapter())
```

Then in `main.py`: `from tasks import aloha`. Switch via `AF_TASK=aloha_transfer` env or per-call `task="aloha_transfer"` kwarg. The reasoner DAG does NOT change.

## Adding a new reasoner layer

Pick the right router file by cognitive role:

| Router | Add what |
|---|---|
| `reasoners/visual.py` | Anything that consumes keyframe pixels |
| `reasoners/action.py` | Anything that consumes the action/state trajectory |
| `reasoners/temporal.py` | Anything that operates on detected segments or boundaries |
| `reasoners/composer.py` | Cross-modal synthesis, the entry orchestrator |
| `reasoners/dl.py` | Deep Lake operations |

Decorator template:

```python
@router_name.reasoner(tags=["domain", "leaf"])
async def my_reasoner(
    arg1: SomeType,
    arg2: SomeOtherType,
    task_description: str,
    model: str | None = None,           # MANDATORY — every reasoner accepts model override
) -> dict:                              # Return dict so downstream code knows it's a dict
    result = await router_name.ai(
        ("...prompt..."),
        # positional multimodal args (image data URLs, etc.)
        system=("...system..."),
        schema=MyFlatSchema,            # 2-4 attributes
        model=model,                    # propagate
    )
    return result.model_dump()
```

## Compatible models — open-source only

Every LLM call goes through OpenRouter. Defaults are Qwen3-VL (Apache 2.0).
NVIDIA Nemotron models are listed as drop-in alternatives — **not dependencies**.
Swap by changing the `AI_MODEL` env var (or `SCOUT_MODEL` / `EXPERT_MODEL` for
the smart-routing tier).

## Verification ladder

Before declaring done, run the canonical async curl path:

```bash
# 1. Stack up?
curl -s http://localhost:8080/api/v1/health | jq '.status'

# 2. Agent + all reasoners + skills registered?
curl -s http://localhost:8080/api/v1/discovery/capabilities | jq '.capabilities[] | select(.agent_id=="roboscribe-af") | {n_reasoners: (.reasoners|length), n_skills: (.skills|length)}'

# 3. Ingest, then annotate end-to-end
EXEC=$(curl -sS -X POST http://localhost:8080/api/v1/execute/async/roboscribe-af.ingest_corpus_to_deeplake -H 'Content-Type: application/json' -d '{"input": {"n_episodes": 3}}' | jq -r '.execution_id')
# wait, then
EXEC=$(curl -sS -X POST http://localhost:8080/api/v1/execute/async/roboscribe-af.annotate_episode -H 'Content-Type: application/json' -d '{"input": {"episode_id": 0, "n_keyframes": 4}}' | jq -r '.execution_id')
```

Expected: 16 reasoners + 8 skills registered, annotate_episode returns succeeded with non-empty `grain_segments`, and Deep Lake gets the row.

## When something breaks

Common runtime failures and what they actually mean:

| Symptom | Likely cause |
|---|---|
| `'dict' object has no attribute 'X'` in a composer | Forgot to reconstruct Pydantic from `dict` after `app.call`. Use `SomeModel(**dict_result)` |
| `'Branch' object is not iterable` | Hit `for b in ds.branches` — use `for name in ds.branches.names(): ds.branches[name]` instead |
| `'method' object is not iterable` over `ds.branches.names` | `names()` is a method in deeplake 4.6, not a property |
| ValueError in JSON serialization | Returning a numpy scalar / Branch / Dataset object from a reasoner — cast to Python primitive |
| `agent error (500)` with `duration_ms < 100` | Look at agent container logs: `docker compose logs roboscribe-af --tail=60` for the traceback |
| Pipeline timed out | A vision call hung. Use async endpoint, never sync. Drop n_keyframes if model is slow |

  └─ imports: agentfield.Agent, AIConfig
              helpers.skills_router
              reasoners.{smoke,visual,action,temporal,composer,dl}_router
              tasks.pusht (side-effect: register_adapter)

  └─ calls: visual_thread, action_thread, temporal_segmenter,
            cross_modal_verifier, episode_goal_synthesizer,
            commit_annotation_to_branch (skill)

reasoners/visual.py
  └─ calls: object_detector × N (inside scene_describer)
            scene_describer || motion_phase_classifier (inside visual_thread)

reasoners/action.py
  └─ calls: velocity_profile (skill), trajectory_phase_classifier

reasoners/temporal.py
  └─ calls: visual_action_aligner (skill), boundary_judger,
            segment_narrator × N (meta-spawned)

  └─ uses dl_store.{ensure_dataset, ensure_annotation_branch,
                    ingest_episode, commit_annotation, query_tql}
